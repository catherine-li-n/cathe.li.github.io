electron-builder 打包实录
最近使用electron9.x+vue-cli3+nodejs开发了一个很轻量的窗体软件，在打包时遇到了nodejs库的编译问题，经过两天终于解决了，所以把整个过程记录一下

问题产生

先采用一下倒叙描述我遇到的问题，因为程序用了一个蛮小的第三方库，在运行npm run electron:serve的时候是没有任何问题的，build打包之后首先程序跑不起来，打开调试工具之后才发现这个库报错了。
最初我认为是config里没有写入dependencies，但是写了，重装node_modules, 换cnpm为npm重装node_modules, 改源，甚至换别的打包方式，包括electron-rebuild，asar各种操作都试了一下，都没用

经过一晚上的思考，决定从库本身下手，因为这个库的结构很简单，只是三个js文件，所以就把它的功能从node_modules中剥离了出来，用相对路径来require，经过几番尝试，npm run electron:serve下可以跑了，于是开始打包，终于 ，页面可以正常加载了，这个bug算是暂告一个段落。
