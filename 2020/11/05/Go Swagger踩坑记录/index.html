使用Swagger为接口生成文档

安装

go get -u gitHub.com/swaggo/swag/cmd/swag
go get -u github.com/swaggo/gin-swagger
go get -u github.com/swaggo/files
go get -u github.com/alecthomas/template

Swagger 是基于标准的OpenAPI规范进行设计的。只要按照这套规范编写注解或通过扫描代码生成注解，就能生成统一标准的接口文档和一系列Swagger工具。

swag -v

源码分析

明明只是初始化了一个docs包并注册了一个Swagger相关路由，那么Swagger的文档是如何关联上的，在接口上写的注解又到哪里去了呢？
在第一步中，我们初始化了docs包，对应的其实就是docs.go文件。
实际上在初始化docs包时会默认执行init方法。而在init方法中，会注册相关方法，其主体逻辑是swag会在生成时检索项目下的注解信息，然后将项目信息和接口路由信息按规范生成到包全局变量doc中去。
接着在ReadDoc方法中做一些template的模板映射等工作，完善doc的输出。
ginSwagger.WrapHandler（swaggerFiles.Handler）中的WrapHandler被调用后，swag内部会将其默认调用的URL设置为doc.json
而它返回的CustomerWrapHandler中有一个switch-case来处理path
